## Bottom line

This document is Sean's complete build reference for replacing the pdfmake PDF export
(`src/export/pdf.js`) with a PDFKit-direct implementation that produces formally ISO 14289-1
(PDF/UA-1) compliant output. It covers: (1) a precise content inventory of the current PDF,
(2) exact typographic and colour values to reproduce, (3) pill colour pairs with contrast
ratios, (4) accessibility tagging requirements per section, and (5) a full test data
specification for the Vitest integration test.

The brief is at `.claude/work/013-iccc-pdf-ua/brief.md`. These requirements are subordinate
to the brief; where they conflict, the brief takes precedence and the discrepancy should be
raised with Sonja before proceeding.

## 3. Status pill colours

Pill cells are rendered via `pillCell(status)` in the current code. The function maps a
status string to a key via `statusStyle()`, then looks up the key in `PILL_COLOURS`.

### 3.1 Status-to-key mapping

| Input status string  | Key     |
|----------------------|---------|
| `'PASS'`, `'SAFE'`   | `pass`  |
| `'FAIL'`, `'HIGH'`   | `fail`  |
| anything else         | `warn`  |
| (key override)        | `neutral` — only reached by explicit fallback |

Note: the pdfmake code also has a catch-all `|| PILL_COLOURS.neutral` if `key` is
undefined. In practice, `statusStyle` always returns one of the three named keys.
The `neutral` pill is used in the summary table for the `'NO TEXT'` verdict because
`statusStyle('NO TEXT')` returns `'warn'` — but the brief implies `neutral` might be
more appropriate. Sean should check with Sonja before changing this; for now, replicate
the existing behaviour exactly.

### 3.2 Pill colour pairs

| Key       | Text colour | Fill colour | Stated contrast | WCAG level |
|-----------|-------------|-------------|-----------------|------------|
| `pass`    | `#14532d`   | `#dcfce7`   | 9.1:1           | AAA        |
| `fail`    | `#7f1d1d`   | `#fee2e2`   | 10:1            | AAA        |
| `warn`    | `#663a00`   | `#fef3c7`   | 8.7:1           | AAA        |
| `neutral` | `#4b5563`   | `#f0f2f5`   | 7.6:1           | AAA        |

### 3.3 Pill visual rendering

In pdfmake, a pill is a single-cell borderless table with `fillColor` on the cell, using
`margin: [6, 2, 6, 2]` (6 pt horizontal padding, 2 pt vertical padding) and `fontSize: 8`.

In PDFKit, Sean must replicate this appearance. The recommended technique from the smoke
test brief is: draw the background rectangle as an `Artifact` (decorative), then overlay
the status text as a tagged `P` or `TD` element with the correct text colour. The pill
background is decorative; only the status word carries semantic meaning.

## 4. Accessibility requirements (PDF/UA-1 tagging)

### Tagging principles

The following rules apply throughout:

- **All meaningful text** must be inside a tagged structure element with the correct role:
  H1, H2, H3 for headings; P for paragraphs; TH/TD for table cells; Figure for images.
- **Decorative fills, background colours, and borders** must be marked as Artifacts and
  must not appear inside any structure element.
- **Images with semantic meaning** (preview, clip region, CVD simulations) require a
  Figure element with an alt text string. Swatch images are decorative (colour comparison
  aid only) and are Artifacts.
- **Links** are plain text in this version (Phase 2). The URL must still appear in the
  document content, but it need not be wrapped in a tagged `/Link` structure element.
- The document must carry `/Lang en` in the document catalog. This is set in
  `createDocument()` via `lang: 'en'`.
- The document must carry the `pdfuaid:part` XMP namespace (value `1`) via
  `doc.appendXML()`. Do not pass `subset: 'PDF/UA'` to the PDFKit constructor.

### 4.1 Branded header block

| Content element         | Tag / treatment |
|-------------------------|-----------------|
| Navy fill rectangle     | Artifact        |
| "Image Colour " text    | Part of H1 (or P) tagged element; acceptable to treat as a single tagged block with the full heading text |
| "Contrast Checker" text | Same tagged element as above |
| Tagline text            | P               |

The visual split of the title into white and orange runs is a styling choice. The tagged
text should read "Image Colour Contrast Checker" as a single continuous string so a
screen reader announces the full name.
